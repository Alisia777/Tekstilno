# Altea Portal · GitHub Pages MVP v6

Обновлённый портал Алтея под развёртывание на GitHub Pages.

## Что добавлено в v6

- **командный режим** с Supabase-ready слоем;
- статус синхронизации прямо в topbar;
- кнопки **Загрузить командные данные** и **Отправить в командную базу**;
- закрепление **owner по SKU** из карточки и из контрольного центра;
- **журнал решений** по SKU;
- сохранение в общем контуре:
  - задач,
  - комментариев,
  - решений,
  - owner assignment.

## Что уже есть из прошлых версий

- красивый дашборд бренда;
- лидеры по продажам, оборачиваемости и контенту;
- центр документов;
- вкладка репрайсера;
- форма расчёта заказа;
- контроль задач;
- реестр SKU;
- pipeline новинок;
- ритм weekly / monthly / PMR;
- экран руководителя.

## Режимы работы

### 1. Local

Ничего не нужно настраивать.
Портал работает как раньше — всё хранится в браузере.

### 2. Supabase

Подключается общая база для командной работы.

Подробно смотри:

- `TEAM_BACKEND_SETUP.md`
- `backend/supabase_schema.sql`

## Структура проекта

- `index.html` — портал
- `styles.css` — стили
- `app.js` — фронт и логика командного режима
- `config.js` — локальная конфигурация проекта
- `config.example.js` — шаблон конфигурации
- `data/` — готовые JSON для фронта
- `library/` — рабочие документы demo-режима
- `backend/supabase_schema.sql` — SQL под общую базу
- `TEAM_BACKEND_SETUP.md` — как подключить Supabase
- `SPRINT_1_TEAM_MODE.md` — что закрыто в спринте
- `scripts/build_data.py` — пересборка JSON из новых выгрузок

## Как выложить на GitHub Pages

1. Создай новый репозиторий на GitHub.
2. Залей содержимое этой папки в корень репозитория.
3. Открой repo → **Settings** → **Pages**.
4. В блоке **Build and deployment** выбери:
   - **Source**: `Deploy from a branch`
   - **Branch**: `main`
   - **Folder**: `/(root)`
5. Нажми **Save**.

## Как включить командную синхронизацию

1. Создай проект в Supabase.
2. Выполни `backend/supabase_schema.sql`.
3. Заполни `config.js`.
4. Поставь:

```js
teamMode: 'supabase'
```

5. Открой портал.
6. Если у тебя уже есть локальные задачи — нажми **Отправить в командную базу**.

## Как обновлять данные из выгрузок

```bash
python scripts/build_data.py --source-dir /путь/к/выгрузкам --output-dir ./data --brand-filter Алтея
```

После пересборки закоммить обновлённые JSON в репозиторий.

## Ограничения текущей версии

- роли пока не разграничены на уровне интерфейса;
- для быстрого старта предполагается anonymous auth в Supabase;
- документы в `library/` на GitHub Pages будут публичными;
- рабочие Google Sheets / OneDrive файлы пока не подключены live;
- авто-сигналы строятся из JSON-среза, а не из live-данных.

3. Live-подключение рабочих файлов.
