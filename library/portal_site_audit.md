# SITE AUDIT · v6

## Что проверено

- `app.js` проходит синтаксическую проверку через `node --check`
- `build_data.py` компилируется без ошибок
- структура репозитория собрана под GitHub Pages
- фронт работает в двух режимах:
  - local
  - supabase-ready

## Что уже можно использовать

- read-only дашборд бренда
- контроль задач
- карточка SKU
- закрепление owner по SKU
- журнал решений
- центр документов
- форма расчёта заказа
- витрина репрайсера

## Что стало лучше в v6

- появился общий слой для задач / комментариев / решений / owner
- есть явный статус синхронизации
- можно быстро пушить локальный контур в общую базу
- можно тянуть общий контур обратно в портал

## Что ещё не закрыто полностью

- роли и права доступа пока не разнесены по интерфейсу
- live-подключение рабочих файлов ещё не включено
- документы в `library/` остаются demo-решением и не годятся для публичного боевого режима
- уведомления и история изменений пока не вынесены в отдельный слой

## Рекомендация по боевому запуску

1. Подключить Supabase
2. Перевести документы в Drive / SharePoint
3. Подключить рабочие Google Sheets / OneDrive файлы
4. Включить авторизацию по рабочей почте
5. После этого переводить домен в боевой контур
