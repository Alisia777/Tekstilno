# Manager Journal Site v6

Версия под ваш текущий процесс:

- **Анастасия** — WB
- **Ирина Паламарук** — Ozon
- **Координатор поставок** — отдельная роль без пакета артикулов

## Что изменено

- добавлен **раздельный выбор площадки**: WB / Ozon / Поставки;
- выбор менеджера теперь **фильтруется по площадке**;
- добавлены **рабочие фильтры по пакету артикулов**: приоритет, статус выполнения, поиск;
- обновлён список действий по артикулам;
- ежедневные инструкции и критичность вынесены в отдельный файл **`data/daily-article-overrides.json`**;
- скрыта кнопка общей таблицы, пока не заполнены `appsScriptUrl` и `googleSheetUrl` в `config.js`;
- включён `siteVersion` в `config.js`, чтобы GitHub Pages быстрее подтягивал обновлённые JSON и JS.

## Откуда взяты данные

- план и месячные показатели:
  - `Оцифровка в руб  План продаж WB 2026.xlsx`
  - `План продаж OZON 2026.xlsx`

  - из комментариев и статусов по товарам в этих файлах;
- ежедневные задачи по ролям:
  - `data/daily-tasks.json`

## Что редактировать каждый день

### 1. Базовый план по артикулам
Файл: `data/article-plan.json`

Туда лучше не лезть ежедневно. Это базовый план и месячные цифры.

### 2. Ежедневные задачи по роли
Файл: `data/daily-tasks.json`

Тут обновляете задачи уровня менеджера:
- что сделать по пакету;
- что проверить по цене/марже;
- что передать координатору;
- итоговые задачи на день.

### 3. Ежедневные инструкции по конкретным артикулам
Файл: `data/daily-article-overrides.json`

Это главный файл для ежедневных правок. В нём можно менять:
- `priorityBucket` — `critical` / `urgent` / `planned`;
- `priorityReason` — почему артикул в приоритете;
- `managerTask` — что именно сделать сегодня;
- `sourceComment` — комментарий от руководителя / из анализа.

## Быстрый запуск

1. Полностью замените файлы в GitHub-репозитории содержимым этой папки.
2. Дождитесь публикации GitHub Pages.
3. Один раз сделайте **Ctrl+F5**, чтобы браузер не держал старый `app.js`.
4. При необходимости позже заполните `appsScriptUrl` и `googleSheetUrl` в `config.js`.

## Что не потеряется при обновлении

- входные данные в JSON — остаются в GitHub и в истории коммитов;
- если подключён Google Apps Script, выгруженные отчёты остаются в Google Sheet;
- локальная история браузера остаётся отдельно, потому что сайт использует новый префикс `manager-journal-v6`.
