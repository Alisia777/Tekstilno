# Текстильно — сайт задач по артикулам, поставкам и контролю (v9)

В этой версии сайт переделан под новую логику:

- отдельные вкладки **WB / Анастасия**, **Ozon / Ирина Паламарук**, **Поставки**, **Руководитель**;
- менеджеры **не заносят план и факт руками**;
- по каждому артикулу сайт сам показывает:
  - задачу по артикулу;
  - причину фокуса;
  - плановые метрики по месяцу;
  - фактические метрики из последней выгрузки;
- менеджеру остаётся только:
  - выбрать статус;
  - оставить комментарий;
  - сдать пакет.

## Что именно подтянуто автоматически

### План
Плановые метрики подтянуты из:
- `Оцифровка в руб  План продаж WB 2026.xlsx`
- `План продаж OZON 2026.xlsx`

Сейчас в JSON сохранены месячные планы **январь–сентябрь 2026**.
Для выбранной даты сайт берёт соответствующий месяц и считает:
- план заказов / день;
- план выручки / день;
- план маржи / день.

### Факт
Фактические метрики подтянуты из:
- `wb_ozon_0403.xlsx`

Для WB показываются:
- факт заказов / день;
- факт выручки 7д / день;
- остаток на МП;
- покрытие в днях;
- потребность в допоставке.

Для Ozon показываются:
- факт заказов / день;
- факт net 34д / день;
- остаток на МП;
- покрытие в днях;
- потребность в допоставке.

## Честно по совпадению с планом

Прямое совпадение строк плана найдено не по всем артикулам:

- **WB:** 236 из 266
- **Ozon:** 122 из 175

Для строк без совпадения сайт всё равно показывает задачу, приоритет и фактические метрики — просто без плановой части.

## Что внутри

- `index.html` — интерфейс сайта.
- `styles.css` — стили.
- `app.js` — вся клиентская логика.
- `config.js` — настройки сайта.
- `data/article-plan.json` — пакеты артикулов + план/факт.
- `data/cluster-reference.json` — строки по кластерам / складам.
- `apps-script/Code.gs` — Apps Script для общей видимости руководителю.

## Как обновить сайт

1. Полностью заменить содержимое репозитория файлами из этой версии.
2. Закоммитить изменения.
3. Дождаться обновления GitHub Pages / вашего домена.
4. Открыть сайт и сделать **Ctrl+F5**.

## Что стало лучше для руководителя

Во вкладке **«Руководитель»** теперь видно:
- кто что сдал;
- с какой площадки;
- на какую дату;
- что именно было внутри отправки;
- какие строки отмечены как блокер / нужна помощь.

Пока `Apps Script` не подключён, это работает:
- в текущем браузере;
- через импорт/экспорт JSON.

После подключения `Apps Script` журнал становится общим для всех устройств.

## Как включить общий журнал

1. Создать Google Sheet.
2. Открыть `Расширения → Apps Script`.
3. Вставить код из `apps-script/Code.gs`.
4. Развернуть как `Web app`.
5. Вписать URL в `config.js`:

```js
window.APP_CONFIG = {
  ...,
  appsScriptUrl: "https://script.google.com/macros/s/ВАШ_URL/exec",
  googleSheetUrl: "https://docs.google.com/spreadsheets/d/ВАШ_ID/edit"
};
```

## Логика работы по вкладкам

### WB / Анастасия
- сайт сам подставляет текущий пакет;
- сайт сам показывает план/факт по каждому артикулу;
- менеджер только выбирает статус и пишет комментарий.

### Ozon / Ирина Паламарук
- та же логика;
- дополнительно в факте видно net 34д / день.

### Поставки
- координатор выбирает артикул;
- сайт поднимает строки по кластерам / складам;
- считает рекомендованную отгрузку и оборачиваемость;
- координатор заполняет:
  - что уже в поставке;
  - что в производстве;
  - что в закупке;
  - план отгрузки;
  - ETA;
  - комментарий.

### Руководитель
- видит журнал всех сданных пакетов и форм поставок;
- открывает детали по любой записи;
- может выгружать и импортировать JSON-бэкап.
