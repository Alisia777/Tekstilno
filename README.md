# Тетрадь менеджера — версия 2 под GitHub Pages

Это уже не “просто форма”.  
В этой версии сайт собран под ваш процесс ежедневной отчётности:

- менеджер видит **задачи на день**;
- отмечает **статус выполнения** и пишет **комментарий**;
- ведёт **тетрадь действий** прямо на сайте;
- вносит **факт по заказам, марже и выручке**;
- видит **отклонение от плана**;
- сохраняет **локальную историю**;
- при подключении Apps Script отправляет всё в **Google Sheet**.

---

## Что было не так в прошлой версии

Я специально не прячу это:

1. сайт был слишком общий и не отражал ваш реальный процесс;
2. не было слоя **ежедневных задач**;
3. не было **тетради действий**;
4. не было понятного блока **план / факт / отклонение**;
5. в папке сайта не хватало `styles.css` и `app.js`, хотя `index.html` на них ссылался.

То есть предыдущая сборка была и концептуально сырой, и технически неполной.

---

## Что теперь внутри

- `index.html` — основной интерфейс сайта
- `styles.css` — нормальная визуальная часть
- `app.js` — логика задач, тетради действий, плана, истории, автосохранения и отправки
- `config.js` — настройки компании, ссылки на Google Sheet и Apps Script
- `data/daily-data.json` — ежедневные задачи и план до подключения Google Sheets как источника
- `.nojekyll` — чтобы GitHub Pages не ломал статический сайт
- `apps-script/Code.gs` — backend для отправки в Google Sheet
- `apps-script/README.md` — короткая инструкция по разворачиванию backend

## Как это работает сейчас

### 1) GitHub Pages показывает сайт

Сайт полностью статический и готов для публикации в GitHub Pages.

### 2) Файл `data/daily-data.json` хранит задачи и план

Пока Google не подключён как источник задач, ежедневные задачи и план лежат прямо в репозитории.

Именно там сейчас меняются:

- задачи на день;
- план по заказам;
- план по марже;
- план по выручке;
- приоритеты и комментарии к задачам.

### 3) Менеджер заполняет отчёт

Менеджер:

- открывает свою личную ссылку;
- видит задачи;
- проставляет статусы;
- ведёт тетрадь действий;
- заполняет факт по цифрам;
- сдаёт отчёт.

### 4) Если Apps Script подключён — отчёт уходит в Google Sheet

Без Apps Script сайт всё равно работает и хранит локальные копии в браузере.  
Но для общей командной отчётности нужно подключить `appsScriptUrl`.

## Быстрый запуск на GitHub Pages

1. Создайте новый репозиторий или замените файлы в текущем.
2. Загрузите все файлы из этой папки.
3. Включите **Settings → Pages**.
4. Выберите публикацию из ветки `main` и папки `/ (root)`.
5. Подождите, пока GitHub опубликует сайт.

---

## Что нужно поменять перед запуском

### `config.js`

Заполните хотя бы:

```js
window.APP_CONFIG = {
  organizationName: "Текстильно",
  staticDataUrl: "data/daily-data.json",
  appsScriptUrl: "",
  googleSheetUrl: "",
  clearAfterRemoteSubmit: false,
  maxLocalReports: 80,
  storagePrefix: "textilno_manager_journal_v2"
};
```

### `data/daily-data.json`

Это главный файл на завтра, если вы хотите быстро стартовать.

В нём есть два массива:

- `plans` — план на день;
- `tasks` — задачи на день.

---

## Структура записи плана

Пример:

```json
{
  "date": "2026-04-02",
  "managerName": "Александра",
  "department": "Маркетплейсы",
  "channel": "Wildberries",
  "ordersPlan": 34,
  "marginPlan": 22000,
  "revenuePlan": 125000
}
```

Можно делать:

- общий план на отдел;
- личный план на менеджера;
- общий резервный план со звёздочками `*`.

---

## Структура записи задачи

Пример:

```json
{
  "taskId": "alex-1",
  "date": "2026-04-02",
  "managerName": "Александра",
  "department": "Маркетплейсы",
  "channel": "Wildberries",
  "title": "Проверить карточки с просадкой по выкупу",
  "category": "Карточки",
  "priority": "high",
  "note": "Сфокусироваться на SKU с риском по марже и нестабильным выкупом.",
  "expectedResult": "По каждой проблемной карточке есть комментарий и решение."
}
```

## Как раздавать личные ссылки менеджерам

Сайт сам умеет собирать ссылку вида:

```text
https://ВАШ-САЙТ/?manager=Александра&department=Маркетплейсы&channel=Wildberries&shift=09%3A00-18%3A00
```

По такой ссылке менеджеру уже не нужно вручную заполнять имя, отдел и направление.

## Как подключить общую Google Sheet

1. Создайте таблицу.
2. Откройте `apps-script/Code.gs`.
3. Разверните скрипт как Web App.
4. Вставьте URL `/exec` в `config.js`.
5. Вставьте ссылку на таблицу в `googleSheetUrl`.

После этого:

- каждый отчёт будет сохраняться локально;
- и одновременно уходить в Google Sheet.

---

## Что именно пишется в таблицу

### `DailyReports`
Один отчёт = одна строка.

### `TaskStatusLog`
Каждая задача из отчёта идёт отдельной строкой.

### `ManagerActionsLog`
Каждое действие из тетради менеджера идёт отдельной строкой.
