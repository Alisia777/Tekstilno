# Сайт ежедневных отчётов менеджеров — версия под GitHub Pages

Это готовый статический пакет под быстрый запуск.

## Что внутри

- `index.html` — сама страница сайта
- `styles.css` — стили
- `app.js` — логика формы, черновика, локальной истории и отправки
- `config.js` — сюда вставляются ваши ссылки
- `.nojekyll` — чтобы GitHub Pages не пытался обрабатывать сайт через Jekyll
- `apps-script/Code.gs` — код для Google Apps Script
- `apps-script/README.md` — как развернуть endpoint

## Как запустить завтра

### 1) Создайте репозиторий на GitHub

Самый простой путь:
- создайте новый репозиторий;
- загрузите туда все файлы из этой папки;
- откройте **Settings → Pages**;
- в разделе Build and deployment выберите публикацию из ветки `main` и папки `/ (root)`.

### 2) Настройте Google Sheet

1. Создайте таблицу, например `Ежедневные отчёты`.
2. В таблице откройте **Extensions → Apps Script**.
3. Вставьте код из `apps-script/Code.gs`.
4. Разверните как **Web app**.
5. Скопируйте URL `/exec`.

### 3) Заполните `config.js`

Откройте `config.js` и вставьте:

```js
window.APP_CONFIG = {
  organizationName: "Текстильно",
  appsScriptUrl: "ВСТАВЬТЕ_СЮДА_URL_APPS_SCRIPT",
  googleSheetUrl: "ВСТАВЬТЕ_СЮДА_ССЫЛКУ_НА_ТАБЛИЦУ",
  googleSheetEmbedUrl: "",
  clearAfterSubmit: true,
  maxLocalReports: 40
};
```

### 4) Залейте изменения в GitHub

После этого сайт начнёт работать как общая форма.

## Как раздавать личные ссылки менеджерам

Сайт умеет подставлять имя и отдел из URL.  
Пример:

```text
https://ВАШ_САЙТ/?manager=Анна%20Смирнова&department=Продажи&shift=09%3A00-18%3A00
```

Можно сделать по одной ссылке на каждого менеджера и кинуть в чат.

## Что важно понимать

### 1. GitHub Pages — это статический сайт

Здесь нет своего сервера и базы.  
Поэтому данные пишутся через Google Apps Script в Google Sheet.

### 2. Публичность

Если вы публикуете сайт на GitHub Pages через public repo, код сайта будет публичным.

### 3. Встраивание Google Sheet

Если вы используете `googleSheetEmbedUrl`, таблицу придётся **публиковать в веб**.  
Это подходит только для read-only публичного просмотра.  
Если данные не должны быть публичными — оставьте `googleSheetEmbedUrl` пустым и открывайте обычную таблицу по кнопке `Открыть Google Sheet`.

## Какие поля менять, если захотите адаптировать шаблон

### В HTML

Измените поля в `index.html` внутри `<form id="reportForm">`.

### В JS

Если добавили новые поля, проверьте:
- `copyableFields` в `app.js`
- `exportLocalCsv()` в `app.js`

### В Apps Script

Добавьте те же поля в массив `HEADERS` и в функцию `normalizePayload_()`.

## Подсказка по редкому, но полезному сценарию

Если интернет на устройстве менеджера отвалится, сайт всё равно хранит локальную копию в браузере.
То есть форма не пропадёт бесследно, даже если общая отправка в таблицу не подтверждена.
