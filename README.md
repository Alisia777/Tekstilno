# Tekstilno Portal v25 Full Fixed

Что исправлено в этой сборке:

- Возвращён **полный пакет**, а не обрезанный patch: внутри снова есть `config.js`, `nina.css`, `data/` и `backend/`.
- Исправлена причина «вечной загрузки матрицы»: раньше в архиве не хватало файлов конфигурации и данных.
- В матрице и в таблицах снова видно **WB артикул**.
- В разделе задач добавлен отдельный столбец **WB арт.**, чтобы Вартану было проще ориентироваться.
- В поставках внутри портала снова работает встроенная матрица `nina.html`.

## Что лежит в архиве

- `index.html` — основной портал
- `portal.css`, `portal.js` — основной контур
- `nina.html`, `nina.css`, `nina.js` — полноэкранная матрица кластеров и поставок
- `shared-storage.js` — общий слой хранения для preview / backend
- `config.js` — конфигурация
- `data/` — план, история, матрица Нины, demo reports
- `backend/` — SQL и инструкция для Supabase

## Как обновить

1. Заменить **репозиторий целиком** этой сборкой.
2. Не удалять `CNAME`, если он уже лежит в репозитории.
3. После публикации сделать `Ctrl+F5`.
4. Проверить:
   - `/`
   - `/nina.html`
   - раздел `Кластера и поставки`
   - колонку `WB арт.`

## Хранение отчётности

По умолчанию в `config.js` включён `session-preview`.
Чтобы отчётность и заявки были видны всем:

1. Выполнить SQL из `backend/portal_shared_storage_supabase.sql`.
2. В `config.js` переключить:
   - `backend.provider = "supabase"`
   - заполнить `backend.supabaseUrl`
   - заполнить `backend.supabaseAnonKey`
