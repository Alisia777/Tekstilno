# Tekstilno Portal v22 · UX Review

Это обзорная сборка не для боевой отчетности, а для проверки удобства портала как продукта.

## Что внутри
- `index.html` — новый каркас портала с контурами: задачи, поставки, контроль, отчеты.
- `nina.html` — полноэкранная матрица поставок.
- `portal.js` / `portal.css` — логика и новый интерфейс review-версии.
- `config.js` — точки входа в данные.
- `data/article-plan.json` — пакеты по артикулам и задачи.
- `data/history-data.json` — история по дням для графиков.
- `data/nina-cluster-dashboard.json` — кластерные данные для поставок.
- `data/demo-reports.json` — демонстрационные записи журнала.
- `ux_review_checklist.md` — как проверять удобство и что отмечать.

## Что важно честно
Сохранение здесь пока `demo-local`: все записи видны в журнале, но живут в браузере. Это нужно именно для UX-проверки, чтобы понять структуру маршрутов и экранов до подключения Supabase.

## Как смотреть
1. Открыть `index.html`.
2. Переключать роль слева: руководитель / менеджер WB / менеджер Ozon / координатор.
3. Менять рабочую дату и платформу сверху.
4. Проверить сценарии из `ux_review_checklist.md`.
5. Для глубокой работы по поставкам открыть `nina.html`.

## Что проверять в первую очередь
- Понятно ли, какой экран кому открывать первым.
- Видно ли, что именно было сохранено.
- Не смешиваются ли задачи, поставки и контроль продаж.
- Понятно ли, куда идти по ограниченному остатку и марже.
