## Что в архиве

- `index.html`, `portal.js`, `portal.css` — основной портал
- `nina.html`, `nina.js`, `nina.css` — матрица кластеров и поставок
- `shared-storage.js` — слой хранения preview / backend
- `config.js` — конфигурация
- `data/` — обновлённые JSON из последних сводов
- `backend/` — SQL и пошаговая синхронизация с Supabase
- `exports/` — проверочные CSV по фокусу Вартана

## Как обновить

1. Заменить **репозиторий целиком** этой сборкой.
2. Не удалять `CNAME`, если он уже есть в репозитории.
3. После публикации сделать `Ctrl+F5`.
4. Проверить:
   - `/`
   - `Задачи менеджеров`
   - `Продажи, маржа и PnL`
   - `Кластера и поставки`
   - `/nina.html`

## Хранение отчётности

По умолчанию в `config.js` всё ещё стоит `session-preview`.
Чтобы статусы, комментарии, заявки и ручные данные были видны всем:

1. Выполнить SQL из `backend/portal_shared_storage_supabase.sql`.
2. Открыть `backend/supabase_sync_steps_v28.md`.
3. В `config.js` переключить:
   - `backend.provider = "supabase"`
   - заполнить `backend.supabaseUrl`
   - заполнить `backend.supabaseAnonKey`

## Ограничение по PnL

В последних двух сводах есть корректные продажи, цена и маржа, но **нет отдельного полного PnL**.
Поэтому в этой сборке `PnL` для контроля считается как **proxy от маржинальной прибыли**,
пока не подключим полный cost-sync из файлов доходов/расходов.
