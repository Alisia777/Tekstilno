# Tekstilno Portal v24

Что изменено в этой сборке:

- В раздел `Кластера и поставки` возвращена полная форма поставок.
- Внутри портала снова встроена полная матрица `nina.html` через iframe.
- Сверху в поставках оставлены быстрые сводки: топ дефицитов и потребность по кластерам.
- Добавлен контур `Продажи, маржа и PnL`:
  - план маржи / день
  - факт маржи / день
  - план PnL / день
  - факт PnL / день
  - таблица сравнения WB vs Ozon: куда выгоднее грузить
  - красная зона по плану, марже и PnL
- В `nina.html` добавлены URL-параметры:
  - `?embed=1` — упрощённый режим для встраивания
  - `?platform=WB` / `?platform=Ozon` — стартовая площадка

## Что лежит в архиве

- `index.html` — основной портал
- `portal.css` — стили основного портала
- `portal.js` — логика основного портала
- `nina.html` — полноэкранная матрица кластеров и поставок
- `nina.css` / `nina.js` — логика и стили матрицы
- `shared-storage.js` — слой хранения
- `config.js` — настройки
- `data/` — данные плана, истории, отчётов, кластерной матрицы
- `backend/` — SQL и инструкция для общего хранения в Supabase

## Как проверить

1. Полностью заменить репозиторий этой сборкой.
2. Не удалять `CNAME`, если он уже есть в репозитории.
3. После публикации сделать `Ctrl+F5`.
4. Проверить:
   - `/`
   - `/nina.html`
   - раздел `Кластера и поставки`
   - раздел `Продажи, маржа и PnL`

## Хранение отчётности

Сейчас по умолчанию включён режим `session-preview`, чтобы интерфейс был живым сразу.
Для общего журнала и видимости всей команды нужно:

1. Выполнить SQL из `backend/portal_shared_storage_supabase.sql`.
2. В `config.js` переключить:
   - `backend.provider = "supabase"`
   - заполнить `backend.supabaseUrl`
   - заполнить `backend.supabaseAnonKey`

После этого статусы задач, журнал и заявки будут общими.
