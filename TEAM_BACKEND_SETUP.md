# Командный режим v6 · как подключить общую базу

## Что уже сделано в коде

В v6 портал умеет работать в двух режимах:

- **local** — всё живёт в браузере, как и раньше;
- **supabase** — задачи, комментарии, журнал решений и закрепление owner синхронизируются между людьми.

Во фронте уже есть:

- статус подключения в topbar;
- кнопка **Загрузить командные данные**;
- кнопка **Отправить в командную базу**;
- сохранение owner по SKU;
- журнал решений;
- общий задачник и апдейты.

## Быстрый запуск через Supabase

### 1. Создай проект

Создай новый проект в Supabase.

### 2. Включи быстрый auth для MVP

Для самого быстрого старта используй **Anonymous Auth**.
Это даст каждому браузеру анонимную authenticated-сессию и позволит синхронизировать данные без отдельной формы логина.

Когда захотите боевой режим с ролями, потом спокойно перейдёте на email magic link или SSO рабочей почты.

### 3. Прогони SQL

Открой SQL Editor и выполни файл:

- `backend/supabase_schema.sql`

Он создаст таблицы:

- `portal_tasks`
- `portal_comments`
- `portal_decisions`
- `portal_owner_assignments`

и включит RLS-политики для authenticated users.

### 4. Заполни config.js

Открой файл `config.js` и выставь так:

```js
window.APP_CONFIG = {
  brand: 'Алтея',
  teamMode: 'supabase',
  teamMember: {
    name: 'Виктория',
    role: 'Бренд-лид'
  },
  supabase: {
    url: 'https://YOUR_PROJECT.supabase.co',
    anonKey: 'YOUR_SUPABASE_ANON_KEY',
    auth: 'anonymous'
  }
};
```

Если не хочешь сразу хранить реальные значения в репозитории, оставь `config.js` локально и не коммить его.

### 5. Открой портал

Когда портал загрузится:

- сверху появится статус подключения;
- станет доступна кнопка **Загрузить командные данные**;
- станет доступна кнопка **Отправить в командную базу**.

### 6. Если у тебя уже есть локальные задачи

Нажми:

- **Отправить в командную базу**

Это закинет текущий локальный JSON в Supabase.

После этого другие участники смогут нажать:

- **Загрузить командные данные**

и увидеть тот же контур задач.

## Как перейти из MVP в боевой режим

Следующий слой после v6:

1. заменить anonymous auth на email / рабочую почту;
2. добавить таблицу профилей и роли;
3. ограничить update/delete по ролям;
4. вынести документы в Google Drive / SharePoint;
5. настроить live-загрузку рабочих Excel/Sheets.

## Что сейчас синхронизируется

- ручные задачи;
- комментарии / апдейты;
- решения по SKU;
- закрепление owner по SKU.

## Что пока остаётся локальным

- авто-сигналы, которые строятся из JSON-данных фронта;
- сам read-only аналитический слой;
- документы из папки `library/`.
