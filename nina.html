Как считаем потребность

  1. Доля кластера = заказы 7д кластера / заказы 7д артикула.
  2. Сезонный план / день = план дня текущего месяца × доля кластера.
  3. Спрос / день для расчёта = максимум из фактического среднего дня и сезонного плана.
  4. Доступно = запас в кластере + в пути + производство + закупка.
  5. Реком. отгрузка = max(0, спрос/день × целевые дни − доступно).

Как учли сезонность

Сезонность не «на глаз». Для артикула берется план/день по месяцам и рассчитывается коэффициент сезона:

Коэф. сезона = план/день текущего месяца ÷ средний план/день по доступным месяцам.

Если нужно, в форме можно задать seasonality override и пересчитать товар вручную.

Логика работы Нины

  • Смотреть матрицу и фильтр «Только где есть потребность».
  • Если часть товара уже едет / в производстве / в закупке — заносить это в форму.
  • После этого формировать заявки на товар по кластерам.
