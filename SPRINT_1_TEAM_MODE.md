# Sprint 1 · Командный контур

## Что закрыто в v6

- Supabase-ready слой для командной синхронизации;
- общий контур для задач, комментариев и owner;
- журнал решений по SKU;
- быстрый экран закрепления owner в контрольном центре;
- topbar-статус командного режима;
- кнопки pull / push для командной базы.

## Что ещё доделать следующим спринтом

1. Профили пользователей и роли.
2. Email / рабочая авторизация вместо anonymous auth.
3. Уведомления по просрочкам и waiting decision.
4. История изменений по задачам.
5. Интеграция рабочих файлов из Google Drive / SharePoint.
6. Автопересборка JSON по расписанию.

## Что это даёт уже сейчас

- один SKU = один owner;
- один задачник = один слой контроля;
- решения не теряются в чатах;
- портал перестаёт быть только витриной и начинает работать как общая операционная система.
